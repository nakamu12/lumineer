## 2. ヒアリング質問リスト

### 戦略的質問（必ず聞く — 方向性を決める）

#### A. 成功基準・評価
- 「パネルプレゼンでは、技術の深さと幅のどちらを重視しますか？」
- 「Requirement 1を深く完成させるのと、Requirement 2まで広くカバーするのと、どちらが評価されますか？」
- 「デモで"これは見たい"というシナリオはありますか？」

#### B. スコープ確認（MoSCoWの叩き台を見せながら）
- 「仮のMoSCoW分類を作ってきました。Must Haveの認識は合っていますか？」
- 「マルチモーダル入力（音声・画像）はWon't Haveとしましたが、この判断はどう思いますか？」
- 「フロントエンドはシンプルで良いか、それともUI/UXも評価対象ですか？」

#### C. ユーザー像
- 「"学生"のイメージをもう少し具体的にすると、どんな人を想定していますか？」
- 「学生以外（例えばアカデミックアドバイザー）も使う想定はありますか？」

#### D. データ
- 「Courseraデータセットのみで良いですか？データの前処理の工夫も評価ポイントですか？」
- 「データの品質問題（欠損値、型の不整合）への対処はどの程度求められますか？」

#### E. 技術的制約
- 「使用するLLMに制約はありますか？（OpenAI, Claude, OSS等）」
- 「デプロイ環境の指定はありますか？ローカルデモで良いですか？」

#### F. 予算・コスト
- 「月額の運用予算はどの程度を想定していますか？」
- 「こちらの試算では、無料枠を活用して月額 ~$30（PoC）〜 $100（小規模商用）で運用可能な構成を考えています。この範囲は妥当ですか？」
- 「LLM APIの利用コスト（トークン課金）について、上限の目安はありますか？」

##### 仮置き予算案（ヒアリング時に提示）

| プラン | 月額 | 構成概要 |
|---|---|---|
| **Minimal（PoC）** | ~$6/月 | GCP Cloud Run + Firebase Hosting + Qdrant Cloud。全て無料枠。LLM API費のみ |
| **Standard（小規模商用）** | ~$30/月 | Cloud Run有料枠（min-instances=1でコールドスタート回避）、Qdrant Cloud有料枠 |
| **Growth（将来拡張）** | ~$100/月 | GCE VM or GKE、冗長化、Langfuse Cloud、バックアップ |

### 実装的質問（後で確認でOK）

- チャンキングサイズの推奨値
- 具体的な評価メトリクスの閾値（Faithfulness > 0.8 等）
- フロントエンドのフレームワーク指定
- CI/CDの要否
- チーム内の役割分担（チーム制の場合）

## 3. ヒアリング時の心構え（FDE Business Skills §3, §7参照）

### 準備チェックリスト
- [ ] 仕様書を読み込み、不明点をリストアップ済み
- [ ] 仮アーキテクチャを1案持っている（議論の叩き台）
- [ ] MoSCoW仮分類を作っている（上記）
- [ ] 質問を「戦略的」と「実装的」に分類済み（上記）

### インタビューのコツ
- **オープンエンドな質問**で始める（「どう思いますか？」> 「〇〇で良いですか？」）
- **仮説を持って臨む**（「こう考えていますが合っていますか？」形式）
- **沈黙は3秒待つ** — 深い答えが出ることが多い
- **メモを取る** — 後で要件に変換する

### ヒアリング後のアクション
- [ ] ヒアリング結果をもとにMoSCoWを確定版に更新
- [ ] ユーザーストーリーを3-5個作成
- [ ] アーキテクチャの叩き台を更新
- [ ] 次回のヒアリング/レビューの日程を確認
